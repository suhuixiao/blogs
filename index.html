前端性能优化--避免回流和重绘

Posted on 2018-12-17

先了解浏览器是如何渲染的

用户请求的资源通过浏览器网络层到达渲染引擎后。渲染工作就会开始。

第一步：浏览器解析HTML文档和解析CSS样式表形成DOM树和CSSOM树

第二步 : 结合DOM树和CSSDOM树形成render树。也就是我们所说的渲染树。

第三步： 浏览器在render树内对每个render节点进行布局处理，计算出每一个元素的大小和位置。确定其在屏幕上的位置

第四步：绘制。通过遍历render树将实际的像素显示到屏幕上

回流（reflow）与重绘（repaint）的定义

回流(reflow)：当render树中的一部分或者全部因为大小边距等问题发生改变而需要重建的过程叫做回流

重绘(repaint)：当元素的一部分属性发生变化，如外观背景色不会引起布局变化而需要重新渲染的过程叫做重绘

什么情况下会引起回流和重绘

dom树结构变化，比如删除或者添加某个dom

内容改变——比如文本内容变更

激活伪类，如：hover

dom元素几何属性变化，包括margin、padding、height、width、border

获取某些属性：offsetTop、offsetLeft、 offsetWidth、offsetHeight、scrollTop、scrollLeft、scrollWidth、scrollHeight、 clientTop、clientLeft、clientWidth、clientHeight、getComputedStyle。浏览器为取得这些属性的值会触发回流。

浏览器窗口大小变化

总的来说，改变某个元素的背景色、文字颜色、边框颜色等等不影响它周围或内部布局的属性时，会引起重绘；如果影响到了布局，则会引起回流和重绘。

回流一定会伴随着重绘，但是重绘不一定会有回流

一般来说，回流比重绘的代价要高，至于具体的花销跟render树有多少节点需要重新构建有关。

通过以上分析基本可以发现。在前端页面中，性能消耗最大的就是 DOM 操作，而且这部分代码会让整体项目的代码变得难以维护。

React则引入了虚拟DOM（Virtual DOM）的机制

什么是虚拟dom

在React中，render执行的结果得到的并不是真正的DOM节点，结果仅仅是轻量级的JavaScript对象，我们称之为virtual DOM。

虚拟DOM是React的一大亮点，具有batching(批处理)和高效的Diff算法，它会计算出虚拟页面当前版本和新版间的差异，基于这些差异对DOM进行必要的最少更新。

虽然虚拟DOM很强，但是也有缺点。比如：

数据变化从Components->Components-1传递下来，React不会只重渲染Components-1和其父组件，React会以重新渲染Components下的虚拟DOM，就导致了组件没必要的重渲染。

那么，提高React虚拟DOM的效率则是一个优化点。React开发团队也考虑到这个问题，shouldComponentUpdate，这个方法是我们的性能优化切入点。之前提供了 PureRenderMixin 插件，在 React 的最新版本里面，提供了 React.PureComponent 的基础类，而不需要使用这个插件。

Fiddler做抓包分析

Posted on 2018-10-15

Fiddler是什么

Fiddler是一个抓包工具，可以将网络传输发送与接受的数据包进行截获、重发、编辑、转存等操作。也可以用来检测网络安全。

Fiddler抓包原理

Fiddler是通过改写HTTP代理，让数据从它那通过,来监控并且截取到数据。在打开它Fiddler那一瞬间，它就已经设置好了浏览器的代理了。当你关闭的时候，它又帮你把代理还原了。

基本设置

  • 接收数据设置

Fiddler想要抓到数据包，要确保Capture Traffic是开启的状态，在File这一栏选择Capture Traffic。开启后在左下角会有图标显示。

  • HTTPS设置(根据需要)

默认是没开启https支持的，当项目中的接口使用的是https协议时，需要打开该功能。

Tools -> Options

  • 手机端(客户端)设置-需要手机抓包分析时

Tools -> Options -> Connections -> Allow remote computers to connect

保证电脑和手机在同一局域网下，设置手机代理服务器地址为电脑ip和端口号8888（默认）。

用手机浏览器打开http://电脑ip.:8888，下载证书—安装证书

功能介绍

1. Statistics 请求的性能数据分析

如上图，我们能够看到一些基本性能数据：如DNS解析的时间消耗是8ms，建立TCP/IP连接的时间消耗是73ms，Https握手的时间是76ms等信息。

2. Inspectors 查看数据内容

Inspectors分为上下两个部分，上半部分是请求头部分，下半部分是响应头部分。对于每一部分，提供了多种不同格式查看每个请求和响应的内容。图片格式使用 ImageView 就可以看到图片，HTML/JS/CSS 使用 TextView 可以看到响应的内容。Raw标签可以查看原始的符合HTTP标准的请求和响应头…

3. AutoResponder 允许拦截指定规则的请求

AutoResponder允许你拦截指定规则的求情，并返回本地资源或Fiddler资源，从而代替服务器响应。本地替代线上调试代码会比较有用。

Posted on 2018-09-18

什么是XSS攻击

XSS攻击就是跨站脚本攻击 - Cross Site Scripting， 看名字应该简称CSS攻击，但是这样会造成误会，以为是css层叠式样式造成了网站安全，所以改称为XSS攻击。

XSS攻击方式

XSS攻击通常被分为两类：反射型和存储型。

反射型

发出请求时，XSS代码出现在URL中，作为输入提交到服务器端。服务器解析后响应，XSS代码随响应内容一起传回浏览器，最后浏览器解析执行XSS代码。这个过程像一次反射，故叫反射型XSS。

存储型

注入的脚本被永久的存储在了目标服务器中，比如数据库、论坛帖子、访问日志、留言评论等。受害者向服务器请求获取存储的信息时，就获得了这些恶意脚本。

存储型XSS和反射型XSS的差别在于：反射型XSS的插入脚本存储于url，存储型XSS的插入脚本存储在服务器端（数据库，内存，文件系统等），只是下发到页面的数据来源不同，以下就不作演示了。

XSS的防御措施

  • 编码

不能对用户的输入原封不动地保存和展示，要对用户输入的数据进行HTML Entity编码

  • 过滤

移除用户上传的DOM属性，如onerror等

移除用户上传的style节点、script节点、iframe节点

  • 校正

避免直接对HTML Entity解码

使用DOM Parse转换，校正不配对的DOM标签

javascript展示内容时使用textContent或者innerText，不使用innerHTML；

Mongoose的概念

Mongoose是在node.js环境下对mongodb进行便捷操作的对象模型工具。

Mongoose中的三个概念

  • schema ：用来定义表的模版，实现和MongoDB数据库的映射。用来实现每个字段的类型，长度，映射的字段，不具备表的操作能力。
  • model ：具备某张表操作能力的一个集合，是mongoose的核心能力。我们说的模型就是这个Mondel。
  • entity ：类似记录，由Model创建的实体，也具有影响数据库的操作能力。
